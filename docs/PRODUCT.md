# Product

## Concept

Mandaloria will be a community platform with four main areas:

- Plazas: Reddit-like posts and debates.
- Codex Libre: official or curated knowledge.
- Holochat: lightweight Discord-like conversation channels.
- Clans/Houses: social structure, identity, ranks, and belonging.

The four areas are connected through the knowledge cycle: conversations from Plazas, Holochat, or Houses can become reviewed proposals for the Codex Libre.

## Initial MVP

The first version must allow the community to already live within the platform.

Core features:

- Registration and login.
- User profile.
- Main post feed.
- Categories or plazas.
- Create, edit, and delete own posts.
- Comment on posts.
- Basic reactions.
- Save posts.
- Report content.
- Basic roles: user, moderator, administrator.
- Minimum admin panel.
- Basic article library.

## Plazas

Plazas are spaces for persistent conversation.

Examples:

- General
- Philosophy
- Lore
- New member questions
- Debates
- Art and creations
- Announcements
- Off-topic

Each post can have:

- title;
- content;
- author;
- plaza;
- tags;
- comments;
- reactions;
- status;
- creation date;
- last activity date.

Possible statuses:

- published;
- featured;
- closed;
- archived;
- under review;
- deleted.

## Codex Libre

The Codex Libre stores more stable content than a debate. Its goal is to provide free and open access to the essential information of Mandaloria.

Content types:

- principles;
- articles;
- guides;
- frequently asked questions;
- foundational texts;
- community rules;
- recommended resources.

Initially the library can only be edited by administrators or members with the Archivist role.

Distinctive flow:

- propose a conversation for the Codex;
- select allowed sources;
- prepare a draft;
- review privacy, attribution, and quality;
- publish a version;
- link article and conversation in both directions.

Accepted names within the product:

- Codex Libre
- Mandalorian Archive
- Library

## Holochat

The chat should start simple.

Initial features:

- public channels;
- welcome channel;
- general channel;
- questions channel;
- read-only announcements channel;
- basic history;
- message moderation.

Future features:

- channels per clan/house;
- mentions;
- online presence;
- pinned messages;
- real-time notifications.

## Clans and Houses

Clans/houses provide social identity within Mandaloria.

Each clan/house can have:

- name;
- description;
- emblem;
- leaders;
- members;
- own channel;
- internal posts;
- internal ranks.
- mission and areas of responsibility;
- research expeditions;
- Codex topics it helps maintain;
- verifiable badges when it has authority to issue them.

In the first version, clans/houses can exist as a basic entity. Advanced permissions can come later.

## Moderation

Mandaloria needs moderation from the start.

Basic features:

- report posts;
- report comments;
- review report queue;
- close posts;
- feature posts;
- hide content;
- quarantine content;
- restore reversible actions;
- ban or suspend users;
- change roles;
- log moderation actions.

## Admin

The admin panel must allow:

- manage users;
- manage roles;
- manage plazas/categories;
- review reports;
- administer the library;
- manage knowledge proposals;
- configure clans/houses;
- view moderation logs;
- change basic site settings.

Conceptual name for the admin/moderation area: Council.

## Customization

The platform must be prepared to be configurable.

Customizable areas:

- site name and description;
- plazas/categories;
- roles and permissions;
- ranks;
- clans/houses;
- badges;
- reactions;
- visual themes;
- welcome texts;
- community rules.
