# Phase 3 - Moderation and Basic Admin

## Objective

Make Mandaloria manageable and moderatable.

## Reports

- [ ] Report post.
- [ ] Report comment.
- [ ] Report profile.
- [ ] Choose reason.
- [ ] Add description.
- [ ] Prevent abusive duplicate reports.
- [ ] Report queue.

## Post moderation

- [ ] Hide post.
- [ ] Restore post.
- [ ] Delete post as moderator/admin.
- [ ] Lock post.
- [ ] Reopen post.
- [ ] Pin post.
- [ ] Highlight post.
- [ ] Move post.
- [ ] Lock editing.
- [ ] Put in quarantine during investigation.
- [ ] Remove from quarantine and restore allowed state.

## Comment moderation

- [ ] Hide comment.
- [ ] Restore comment.
- [ ] Delete comment as moderator/admin.
- [ ] Pin comment.
- [ ] Lock replies.
- [ ] View edit history if it exists.
- [ ] Put comment and related attachments in quarantine.

## Reversibility

- [ ] Hiding does not destroy content or evidence.
- [ ] Quarantine removes content from feed, search, previews and public storage.
- [ ] Restore preserves traceability.
- [ ] Permanent deletion requires superior permission, reason and confirmation.
- [ ] Appeal is linked to the original action.
- [ ] Evidence has limited access and retention.

## Users

- [ ] Warn user.
- [ ] Suspend user.
- [ ] Ban user.
- [ ] Unban user.
- [ ] View user moderation history.
- [ ] Add internal note.

## Logs

- [ ] Log moderation actions.
- [ ] Log admin actions.
- [ ] View audit logs.
- [ ] Filter logs.

## Security

- [ ] Moderator only acts within their scope.
- [ ] Moderator cannot moderate admins.
- [ ] Strong actions require specific permission.
- [ ] Every sensitive change is audited.

## Done when

- [ ] A user can report.
- [ ] A moderator can resolve reports.
- [ ] An admin can audit actions.
- [ ] A moderator can delete/hide messages and posts according to permissions.
- [ ] Reversible actions can be undone without editing the database manually.
