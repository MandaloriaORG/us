# Phase 7 - Customization, Search and Hardening

## Objective

Make Mandaloria configurable, searchable and more resilient.

## Customization

- [ ] Site settings.
- [ ] Configurable site name.
- [ ] Configurable description.
- [ ] Configurable navigation.
- [ ] Configurable reactions.
- [ ] Per-plaza rules.
- [ ] Custom profile fields.
- [ ] Custom post fields.
- [ ] Configurable initial visual theme.
- [ ] Feature flags.

## Search

- [ ] Search posts.
- [ ] Search comments if applicable.
- [ ] Search articles.
- [ ] Filter by plaza.
- [ ] Filter by tag.
- [ ] Filter by author.
- [ ] Respect permissions in results.
- [ ] Exclude deleted/private content.

## Attachments

- [ ] Avatars.
- [ ] Clan/casa emblems.
- [ ] Images in posts if allowed.
- [ ] Images in Codex if allowed.
- [ ] MIME type limits.
- [ ] Size limits.
- [ ] Separate public/private buckets.

## Hardening

- [ ] Review all RLS policies.
- [ ] Review server-side actions.
- [ ] Review XSS.
- [ ] Review admin permissions.
- [ ] Review rate limits.
- [ ] Review search and private content.
- [ ] Review logs.
- [ ] Review backups.
- [ ] Regression tests.

## Done when

- [ ] Admin can customize the essentials.
- [ ] Search respects visibility.
- [ ] Attachments are safe.
- [ ] Security was reviewed before production.
